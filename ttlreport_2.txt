Modern AI systems increasingly rely on multimodal learning to capture richer contextual understanding. Human emotions are complex and cannot be accurately captured through a single modality. By integrating EEG, GSR, eye-tracking, and facial data, the system achieves more robust and reliable emotion recognition than unimodal systems. Recurrent Neural Networks (RNN) RNNs and models are highly effective for sequential and temporal data modeling . They are widely used in speech recognition, time-series prediction, sentiment analysis, and behavioral modeling due to their ability to capture temporal dependencies. Emotions evolve over time, making temporal modeling essential. RNN layers capture sequential dependencies in emotional transitions, enabling the system to predict future emotional states rather than static labels. Advantages: Excellent for sequential data Captures temporal dependencies Supports long-term behavioral modeling Improves emotion transition understanding Disadvantages: Computationally intensive Vanishing gradient issues (basic RNN) Longer training times Requires large datasets 4. Hidden Markov Model (HMM) Hidden Markov Models are statistical sequence models that represent systems transitioning between hidden states over time. HMMs remain highly relevant for time-series analysis, speech recognition, and behavioral state modeling . While RNN captures temporal patterns, HMM models probabilistic emotional state transitions explicitly. Combining HMM with RNN provides both deep sequential learning and interpretable state transition probabilities. Advantages: Strong probabilistic framework Interpretable transition matrices Effective for temporal state modeling Complements deep learning architectures Disadvantages: Assumes Markov property Limited representation for highly complex dependencies May require careful tuning 5. Unlike traditional emotion recognition systems that classify emotions at a single static moment, this project focuses on understanding how emotional states evolve sequentially during user interactions. By leveraging temporal analysis, the system captures emotional transitions, engagement patterns, and behavioral trajectories, making it highly relevant for intelligent human-computer interaction, adaptive learning systems, mental health monitoring, and personalized user experience applications. The system integrates multiple physiological and behavioral data modalities, including Electroencephalography (EEG) signals, Galvanic Skin Response (GSR) , Eye Tracking data , and Facial Emotion Recognition to build a comprehensive understanding of emotional states. These diverse data streams are synchronized, preprocessed , and transformed into meaningful features before being fused into a unified multimodal representation. Using a hybrid Recurrent Neural Network (RNN) and Hidden Markov Model (HMM) architecture, the platform not only identifies emotional states but also predicts future transitions and clusters users into distinct engagement categories. Unlike conventional emotion detection systems that rely solely on facial expressions or isolated sensor data, this project emphasizes multimodal fusion and temporal dependency modeling to achieve higher predictive accuracy and richer emotional interpretation. The integration of deep learning with probabilistic state transition modeling enables both powerful sequence learning and interpretable behavioral analysis. Developed as part of the Technology Trends Lab (TTL) course, this project demonstrates the practical application of cutting-edge technologies in Artificial Intelligence, Machine Learning, Deep Learning, and Data Analytics. By combining Python, PyTorch /TensorFlow, Scikit-learn, Flask, and advanced visualization tools, the project delivers a robust, real-time, and production-oriented emotion detection framework capable of supporting future innovations in affective computing and intelligent adaptive systems . Problem Statement and Need of Work: Understanding human emotions using AI is complex because emotions are not static—they change continuously over time. Most existing emotion recognition systems rely on a single data source such as facial expressions or physiological signals and provide only momentary predictions. These systems fail to capture the dynamic nature of emotions and often produce inaccurate or incomplete results. The core problems this project addresses are: • Inability of traditional systems to model temporal emotion transitions • Low accuracy due to reliance on single-modality data • Lack of synchronization in multimodal time-series data (EEG, GSR, eye tracking, facial data) • Difficulty in handling noisy, missing, and heterogeneous data • Limited use of advanced models for sequential and probabilistic emotion prediction The need for this project arises from the growing demand for intelligent systems that can understand emotional behavior over time. The Temporal Emotion Detection System solves these problems by combining multimodal data processing, feature fusion, and a hybrid RNN- BiLSTM with HMM model to accurately predict emotion transitions and user engagement patterns. Design and implement a robust data preprocessing pipeline to handle synchronization, noise removal, missing values, and normalization of heterogeneous time-series data. Extract meaningful features from each modality (physiological, behavioral , and visual) to accurately represent emotional states and patterns . Implement an effective multimodal feature fusion strategy to combine information from different data sources into a unified representation for improved prediction accuracy. Develop a hybrid deep learning architecture using RNN and Hidden Markov Models (HMM) to model temporal dependencies and emotional state transitions. Enable prediction of future emotional states and analysis of emotion trajectories over time rather than static classification. Classify users into different engagement levels or behavioral patterns based on their emotional transitions and interaction data. Build a real-time visualization dashboard to display emotion trends, transition patterns, and engagement insights in an interactive manner. Evaluate system performance using appropriate metrics and validate the effectiveness of multimodal and temporal modeling approaches. Create a scalable and extensible framework that can be applied to domains such as adaptive learning, mental health monitoring, and intelligent human-computer interaction systems. Proposed Work The proposed work for the Temporal Emotion Detection System focuses on developing a complete end-to-end multimodal emotion analysis pipeline with temporal modeling and real-time visualization. The key areas of work include: a . Multimodal Data Processing Pipeline The core pipeline handles multiple stages of multimodal data processing. Raw data from EEG, GSR, eye-tracking, and facial emotion sources are collected and synchronized using timestamp alignment (e.g., PSY.csv). The preprocessing phase includes noise removal, handling missing values, normalization, and resampling to a uniform frequency. This ensures consistency across heterogeneous time-series data and prepares it for downstream processing. b . Feature Engineering and Extraction After preprocessing, meaningful features are extracted from each modality. EEG signals are transformed into frequency band features (Alpha, Beta, Gamma, Delta), GSR signals are used to derive physiological indicators such as Skin Conductance Level (SCL) and Skin Conductance Response (SCR), eye-tracking data provides metrics like blink rate and pupil dilation, and facial data is processed into action units and emotion probabilities. These features form the basis for understanding emotional states. c . Multimodal Feature Fusion A complete JWT-based authentication system is implemented using the python-jose library for token generation and passlib with bcrypt for password hashing. Users register with a username, email, and password. Upon successful login, a JWT token valid for 7 days is issued. All API endpoints are protected by a FastAPI dependency (get_current_user) that validates the token on every request. d . Temporal Modeling using RNN-HMM Hybrid The system employs a hybrid modeling approach combining Recurrent Neural Networks (RNN) and Hidden Markov Models (HMM). The RNN captures sequential dependencies and temporal patterns in emotional data, while the HMM models probabilistic state transitions between emotional states. This combination enables accurate prediction of emotion trajectories over time. e . Emotion Prediction and Engagement Analysis The trained model is used to predict emotional states at each time step and analyze transitions across the interaction timeline. Based on these patterns, users are grouped into different engagement categories or behavioral archetypes. This helps in understanding user involvement, attention levels, and emotional responses. System Architecture The Temporal Emotion Detection System follows a layered architecture designed to handle multimodal data processing, temporal modeling , and real-time visualization. The system consists of sequential modules starting from data acquisition to final output generation, as illustrated in the architecture diagram. Input Layer (Multimodal Data Sources) The system begins with the collection of multimodal data from various sources, including EEG signals, GSR data, eye-tracking data, and facial emotion data. Each modality captures different aspects of human emotional and physiological behavior , providing a comprehensive input for analysis. These data streams are heterogeneous in nature and may have different sampling rates and formats. Data Preprocessing Layer The preprocessing layer ensures that all input data is clean, synchronized, and suitable for analysis. Timestamp synchronization (using files such as PSY.csv) aligns all modalities into a unified timeline. Additional preprocessing steps include data cleaning, handling missing values, noise removal, resampling/ downsampling , and normalization. This layer is critical for improving data quality and ensuring consistency across modalities . Feature Engineering Layer In this layer, meaningful features are extracted from each modality. EEG signals are converted into frequency band features such as Alpha, Beta, Gamma, and Delta. GSR data provides physiological indicators like Skin Conductance Level (SCL) and Skin Conductance Response (SCR). Eye-tracking data contributes features such as blink rate, pupil dilation, and fixation patterns, while facial data provides action units and emotion probabilities. These features represent the emotional and behavioral state of the user. Feature Fusion Layer The feature fusion layer integrates features from all modalities into a unified representation. Both early fusion (combining raw features) and late fusion (combining model outputs) approaches are used. This integration enhances the robustness and accuracy of the system by leveraging complementary information from different data sources. Modeling Layer : The modeling layer is the core of the system, where temporal analysis is performed. A hybrid approach combining Recurrent Neural Networks (RNN) and Hidden Markov Models (HMM) is used. The RNN captures sequential dependencies and temporal patterns in the data, while the HMM models probabilistic transitions between emotional states. This combination enables accurate prediction of emotion trajectories over time. Output Layer : The final layer generates meaningful insights from the modeled data. Outputs include emotion transition visualization, engagement classification, real-time dashboard display, and adaptive learning insights. These outputs help in understanding user behavior , emotional trends, and engagement levels over time. System Flow Summary : The overall system follows a structured pipeline: Multimodal Inputs → Data Preprocessing → Feature Engineering → Feature Fusion → RNN + HMM Modeling → Outputs (Visualization & Insights) Project Modules Multimodal Data Acquisition Module This module is responsible for collecting and managing input data from multiple sources, including EEG signals, GSR data, eye-tracking data, and facial emotion data. Each modality provides unique information about the user’s physiological and behavioral state. The module ensures proper handling of different data formats and prepares raw inputs for synchronization and preprocessing. Data Preprocessing Module This module handles cleaning and preparing the raw multimodal data for analysis. It performs timestamp synchronization (using files such as PSY.csv) to align all data streams into a common timeline. Additional tasks include noise removal, missing value handling, normalization, and resampling/ downsampling . This ensures that the data is consistent, reliable, and suitable for feature extraction. Feature Engineering Module The feature engineering module extracts meaningful and representative features from each modality. EEG signals are transformed into frequency band features (Alpha, Beta, Gamma, Delta), GSR data is used to compute SCR and SCL, eye-tracking data provides blink rate and pupil dilation metrics, and facial data yields action units and emotion probabilities. These features form the core input for the modeling stage. Feature Fusion Module This module integrates features from different modalities into a unified feature vector. It implements both early fusion (combining features before modeling ) and late fusion (combining outputs from different models). The fusion process enhances system performance by leveraging complementary information from multiple data sources. Modeling Module (RNN-HMM Hybrid) This is the core analytical module of the system. It uses a hybrid approach combining RNN and Hidden Markov Models (HMM). The RNN captures temporal dependencies and sequential patterns in emotional data, while the HMM models probabilistic transitions between emotional states. This module enables accurate prediction of emotion trajectories over time. Emotion Prediction and Analysis Module This module interprets the model outputs to generate meaningful insights. It predicts emotional states at each time step, analyzes transitions between states, and classifies user engagement levels. It may also group users into behavioral clusters based on their emotional patterns. Visualization and Dashboard Module This module presents the results in an interactive and user-friendly format. It generates visualizations such as emotion transition graphs, time-series plots, and engagement analysis charts. A web-based dashboard (using Flask and visualization libraries) allows real-time monitoring and interpretation of emotional data. Matrix with most common emotion transitions Key Implementation Points Multimodal data from EEG, GSR, eye-tracking, and facial emotion sources is synchronized using a common timestamp reference (e.g., PSY.csv) to ensure all signals align on a unified time axis. A dedicated preprocessing pipeline handles noise removal, missing value imputation, normalization, and resampling/ downsampling to make heterogeneous time-series data consistent and model-ready. Feature extraction is performed separately for each modality, such as frequency band extraction for EEG (Alpha, Beta, Gamma, Delta), physiological metrics for GSR (SCR, SCL), behavioral features from eye tracking (blink rate, pupil dilation), and action units/emotion probabilities from facial data. A multimodal feature fusion mechanism is implemented to combine features from all sources into a unified feature vector using early fusion and late fusion strategies. Temporal modeling is achieved using a hybrid architecture combining RNN/ BiLSTM and Hidden Markov Models (HMM), where RNN captures sequential dependencies and HMM models probabilistic state transitions. The system processes data in sequential time windows, enabling analysis of emotion progression and prediction of future emotional states rather than static classification. Model outputs are used to generate emotion transition sequences and classify user engagement levels based on behavioral patterns. Visualization modules generate time-series emotion graphs, transition matrices, and engagement analytics using libraries such as Matplotlib and Plotly. A web-based interface ( Streamlit ) is implemented to display real-time emotional insights and interactive dashboards for user interpretation. Conclusion: The Temporal Emotion Detection System successfully demonstrates the use of multimodal data and temporal machine learning techniques to analyze and predict human emotions over time. Unlike traditional systems, it focuses on capturing emotional transitions using EEG, GSR, eye-tracking, and facial data. The project implements a complete pipeline including data preprocessing, feature extraction, multimodal fusion, and a hybrid RNN/ BiLSTM with HMM model for accurate temporal emotion prediction. It also provides visualization of emotional trends and user engagement. Key achievements include: • End-to-end multimodal emotion detection pipeline • Temporal modeling of emotion transitions • Improved accuracy using multiple data sources • Interactive visualization for analysis Future scope includes adding more modalities like speech, improving models using advanced deep learning techniques, and deploying the system on cloud platforms.
